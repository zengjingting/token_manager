## 概述

Token Dashboard 聚合两个来源的会话数据：

| 数据源 | 工具 | 原始数据位置 |
|---|---|---|
| Claude Code | Claude CLI 会话文件 | `~/.claude/projects/**/*.jsonl` |
| Codex | OpenAI Codex CLI 会话文件 | `~/.codex/sessions/**/*.jsonl` |

当前页面包含两个视图：

- `仪表盘`：展示 Token、费用、模型、项目与会话统计。
- `会话历史`：展示并浏览 Claude + Codex 的会话详情。

刷新机制：

- 仪表盘通过 SSE 每 30 秒自动刷新一次。
- 会话历史不是 30 秒自动刷新；在进入历史页、删除会话、重新搜索或手动切换会话时刷新对应内容。

## 仪表盘

### 时间周期

顶部支持 5 个周期：

- `5小时`：从当前时刻往前 5 小时，按小时展示（`HH:00`）。
- `今日`：聚合为单个当日柱（单点汇总）。
- `3天`：按自然日展示最近 3 天。
- `7天`：按自然日展示最近 7 天。
- `自定义`：按 `since/until` 日期范围展示（日粒度）。

数据来源策略：

- `5小时`：直接扫描本地 JSONL（Claude + Codex）。
- `今日/3天/7天/自定义`：调用 `ccusage`（Claude）和 `ccusage-codex`（Codex）读取聚合结果。

### 统计卡片

页面顶部 4 张卡片均支持 `!` 图标悬浮提示。

1. `总 Token`
   - 公式：`输入 + 输出 + 缓存创建 + 缓存读取`
   - 包含 Claude + Codex。
2. `总费用`
   - 显示合计费用，并在副标题显示 `Claude Code` 与 `Codex` 分项。
3. `缓存命中`
   - 仅按 Claude 计算：`cache_read / (cache_read + cache_creation)`。
4. `模型数`
   - 统计当前周期内出现的模型数量，副标题展示前 2 个模型名。

### Token 分布图

- 分组柱状图，共 6 个系列：
  - Claude 输入 / 输出 / 缓存
  - Codex 输入 / 输出 / 缓存
- 支持勾选显示系列；至少保留一个系列可见。
- Y 轴会按当前可见系列自动缩放（最大值约 `1.1x`）。

### 模型成本分布

- 环形图，按模型展示占比。
- 会随当前时间周期刷新。
- 默认按 `cost` 计算占比；当该周期所有模型成本为 0 时，回退使用 `totalTokens` 作为图值。

### 项目成本分布

- 水平柱状图，展示费用最高的前 15 个项目。
- 仅统计 Claude 项目（Codex 不在此图中）。
- 页面初始化时加载一次，不随时间周期切换自动刷新。

### 会话列表

仪表盘底部会话表（最多 50 条）字段如下：

- 来源：`CLAUDE` / `CODEX`。
- 会话：显示自定义标题（若存在）或会话 ID 缩略。
- Token/输入/输出/缓存/费用/最后活动/模型。

交互：

- 点击会话列可跳转到会话历史并打开对应会话。
- 悬停显示原始会话 ID（Codex 显示终端友好的尾段 ID）。

## 会话历史

### 左侧列表

- 按项目分组展示会话（Claude + Codex 合并分组）。
- 每个项目可展开/收起，默认自动展开第一组。
- 每条会话显示：标题、来源标签、时间、Token、费用。

### 会话操作

每条会话右侧支持：

- `复制`：复制会话 ID（Codex 复制尾段 ID），复制后短暂显示 `✓`。
- `删除`：删除对应本地会话文件，删除前有二次确认。

删除颗粒度：

- Claude：删除 `~/.claude/projects/<project>/<sessionId>.jsonl`
- Codex：删除 `~/.codex/sessions/<sessionPath>.jsonl`（并清理空目录）

删除会减少本地会话文件数量并释放对应存储空间。

### 会话详情

右侧详情区展示：

- 标题（支持重命名）
- 来源、时间、Token、费用、模型
- 消息流（User / Assistant / Tool Use / Tool Result）

消息与展示规则：

- 工具调用消息默认隐藏。
- 点击 `展示工具调用记录 (N)` 可展开/隐藏工具消息。
- 工具消息块可单独折叠/展开。
- 首屏加载 100 条消息，支持 `加载更多`。

### 自定义标题（重命名）

- 标题区悬停显示 `✎`，点击后可直接编辑。
- `Enter` 或失焦保存，`Escape` 取消。
- 自定义标题存储在浏览器 `localStorage`：
  - key: `claude-session-name-{sessionId}`
- 仪表盘会话表与历史列表都会同步显示该自定义标题。

## 搜索

### 历史全局搜索（左上搜索框）

- 输入关键词后点击 `搜索` 或按 `Enter`。
- 点击 `× 清除` 回到完整会话列表。

搜索范围：

- 服务端全文搜索：会话文本消息内容（Claude + Codex）。
- 本地标题搜索：同时检索 `localStorage` 自定义标题。

结果排序：

1. 自定义标题命中的结果优先。
2. 其余按最后活动时间降序。

点击搜索结果会：

- 打开目标会话。
- 自动打开“会话内搜索”并带入关键词。
- 自动跳转到首个命中消息。

### 会话内搜索（详情区右上 `🔍 会话内搜索`）

- 在当前会话内搜索。
- 支持过滤范围：`用户消息`、`Assistant 回复`。
- 点击命中项后自动滚动到对应消息并短暂高亮。
- 正文和结果列表均会高亮匹配词。

## 导出

会话详情右上角 `↓ Markdown` 可导出当前会话为 `.md` 文件。

导出内容：

- 会话元信息（Session、Project、Source、Last Activity、Models、Tokens、Cost）
- 全量消息：用户、Assistant、工具调用、工具结果
- 工具结果在导出中最多保留前 2000 字符

文件名规则：

- `{sessionId前8位}-{原始标题前30字符}.md`
- 文件名中的非法路径字符会被替换为 `-`

下载位置：

- 浏览器默认下载目录（由浏览器设置决定）。

## 当前口径与限制

1. Codex 本地 JSONL 中没有与 Claude 对齐的 `costUSD` 字段，因此“会话历史”里 Codex 会话费用通常为 `0`。
2. `5小时` 模式使用本地 JSONL 实时扫描，Codex 成本在该模式下也通常为 `0`。
3. 项目成本分布目前只统计 Claude 项目，并且按全量历史聚合，不受时间选择器影响。
4. Markdown 导出使用会话原始标题（不是 localStorage 自定义标题）。
